Select which subset of the available FFT bins to read out

Initially we can only read out from a subset of the FFT bins, so this function selects which bins to read out right now This also takes care of writing the selection to the FPGA with the appropriate tweaks

The readout selection is stored to self.readout_selection The FPGA readout indexes is stored in self.fpga_fft_readout_indexes The bins that we are reading out is stored in self.readout_fft_bins

readout_selection
: array of ints
indexes into the self.fft_bins array to specify the bins to read out

set_fft_gain(gain)[source]¶

Set the gain in the FFT

At each stage of the FFT there is the option to downshift (divide by 2) the data, reducing the overall voltage gain by a factor of 2. Therefore, the FFT gain can only be of the form 2^k for k nonnegative

gain: the number of stages to not divide on. The final gain will be 2^gain

kid_readout.roach.valon.check_output(*popenargs, **kwargs)[source]¶

Run command with arguments and return its output as a byte string.

If the exit code was non-zero it raises a CalledProcessError. The CalledProcessError object will have the return code in the returncode attribute and output in the output attribute.

The arguments are the same as for the Popen constructor. Example:

>>> check_output(["ls", "-l", "/dev/null"])
'crw-rw-rw- 1 root root 1, 3 Oct 18  2007 /dev/null\n'

The stdout argument is not allowed as it is used internally. To capture standard error in the result, use stderr=STDOUT.

>>> check_output(["/bin/sh", "-c",
...               "ls -l non_existent_file ; exit 0"],
...              stderr=STDOUT)
'ls: non_existent_file: No such file or directory\n'

kid_readout.roach.valon.find_valons_with_dmesg()[source]¶

Find /dev/ttyUSB* ports with FTDI chips which are probably Valons

Works by parsing the dmesg output and looking for lines containing FTDI (the brand of usb-rs232 chip used) and ‘attached to’ which will contain the ttyUSB* string

returns a list of all unique ports that have FTDI chips. The list is sorted starting with the most recent entry in dmesg. Duplicate ports are ignored (only the most recent entry (line in dmesg) for that port is returned)
